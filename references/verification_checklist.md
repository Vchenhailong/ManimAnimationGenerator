# 验证清单模板

AI 在输出最终代码前必须逐项自检：

## 内容完整性

- [ ] 知识图谱中的每个原子至少对应一个动画步骤。
- [ ] 每个反直觉点都有专门的证伪步骤。
- [ ] 至少包含一个生活/工程应用案例（若用户要求深度）。
- [ ] 教学路径覆盖七个阶段（激活、直观、定义、运算、反直觉、应用、总结），路径文案不展示在动画中。
- [ ] 每个知识原子已标注教材或知识库来源（若为 AI 生成草案则标注待核对）。
- [ ] 若用户提供了教材名称，检查代码注释中是否引用了对应章节。
- [ ] 讲解深度每个新概念满足：定义明确 + 直观解释 + 至少一个例子（必须与用户确认） + 连接前置知识 + 反直觉澄清（如适用）
- [ ] 每个原子元素数量 <= 8
- [ ] 每个原子预估高度 <= 5.5 单位（垂直排列时）
- [ ] 每个原子预估宽度 <= 13.5 单位（安全区宽度，不含溢出缓冲区）
- [ ] 重要公式已独立成原子
- [ ] 已调用 split_atom.py 检查并拆分超长原子
- [ ] **【数理正确性】所有坐标由数学公式推导（中点、交点、切点等），无硬编码**
- [ ] **【数理正确性】几何关系已验证（距离、角度、平行、垂直、相切）**
- [ ] **【数理正确性】物理定律已体现（如适用：牛顿定律、动量守恒、基尔霍夫定律等）**

## 布局底座引用

- [ ] 场景类继承自 `LayoutScene`（来自 `scripts/layout/scene_base.py` ）
- [ ] 使用了 `place_in_safe_area()` 布局方法

## MathTex vs Tex 规范

- [ ] 纯数学公式使用 MathTex，含中文的混合内容使用 Tex
- [ ] MathTex 中无中文字符
- [ ] Tex 中的公式已用 $...$ 包裹

## 布局与视觉

- [ ] 所有教学内容最小 Y >= -2.8，字幕区无其他对象。
- [ ] 元素间距 >= 0.6 或 0.3，无不重叠冲突。
- [ ] 坐标无硬编码，均使用相对定位或数学推导。
- [ ] 若使用坐标系，轴线范围超出图形内容边界。
- [ ] **【坐标参考系】涉及坐标计算的场景已添加 Axes 或 NumberPlane**
- [ ] **【坐标参考系】使用 axes.c2p() 转换坐标，无硬编码 [x, y, z]**
- [ ] X 轴和 Y 轴必须包含正负方向，有轴名标识
- [ ] 主内容已主动定位到顶部居中（Y = 3.2），而非依赖 safe_place 随机放置
- [ ] 使用了 align_to(ORIGIN, UP) + shift(UP \* 2.0) 实现顶部对齐
- [ ] 视频结尾有知识网络图（展示当前概念与前后关联）。
- [ ] 标题区位于 Y = 3.6（如无标题则跳过）
- [ ] 副标题区位于 Y = 3.1 或紧贴标题下方
- [ ] 主内容区从顶部开始排列（首个元素 Y = 3.2）
- [ ] 无主内容时，标题/副标题整体垂直居中
- [ ] 分栏时各栏内容从顶部开始排列（首个元素 Y = 3.2）
- [ ] 分栏时各栏顶部对齐
- [ ] 分栏组合整体水平居中
- [ ] 分栏模式字体大小符合规范（单栏32、两栏30/32/28、三栏28/28/24）
- [ ] 分栏模式宽度比例符合规范：有图形时两栏60/40、三栏30/30/40；无图形时优先均分
- [ ] 单个原子内的 content 元素使用垂直排列（默认）或自动切换为水平/两栏
- [ ] 垂直排列时整体顶部居中（Y = 3.2）
- [ ] 水平排列时整体垂直居中（Y = 0）
- [ ] 内容未超出安全区边界（水平 <= 13.5 单位，垂直 <= 6.8 单位）

## Gate 3：程序化布局校验（代码完成后、渲染前）

> 本节对应「验收门禁」第三道。必须通过 `validate_layout()` 毫秒级程序化检测，不可依赖肉眼渲染判断。

**执行方式**：在场景代码的 `construct()` 方法末尾调用 `self.validate_layout()`，传入场景中所有 mobject 和区域常量。通过标准为返回空列表。

**检测范围**（9 类违规）：

| 违规类型                | 说明                     |
| ----------------------- | ------------------------ |
| REGION_OVERFLOW         | 对象超出区域左右边界     |
| REGION_OUTSIDE          | 对象在区域外侧           |
| REGION_INTRUSION        | 非字幕对象侵入字幕区     |
| ELEMENT_OVERLAP         | 元素两两重叠且无语义相关 |
| STACK_OVERFLOW          | 多元素堆叠总高度超出区域 |
| WIDTH_EXCEEDS           | 单个元素宽度超出列宽     |
| ABNORMAL_SPACING        | 相邻元素间距过密或过稀   |
| OVER_DENSE / TOO_SPARSE | 区域填充率异常           |
| CENTER_OFFSET           | 区域内容重心严重偏移     |

**重叠白名单机制**（语义相关性唯一基准）：使用 `allowed_overlap_patterns=ZC.ALLOWED_PATTERNS` 自动推断以下 13 种语义相关的合法重叠，无需手动配置：

- 物理类（4种）：力箭头↔物体、导线↔元件、场线↔物体、物体↔液体
- 数学/几何类（8种）：顶点↔图形、辅助线↔主图形、角弧↔顶点、标签↔图形、⊥∥符号↔线段、尺寸标注↔线段、切线↔曲线
- 通用（1种）：标注文本↔任意目标

**通过标准**：`validate_layout()` 返回空列表。所有违规必须修复后重新检测，方可进入 Gate 4。

### 电路图元 Don't（负向约束）

> **用途**：电路图代码写成这样 → 画面炸成这样。Agent 必须避免以下任意一条。

#### C-D1：导线 T 型连接处无节点圆点

- **禁止**：三条导线交汇但未画圆点，视觉上无法区分 T 型与 L 型连接
- **必须**：使用 `scripts/physics_graphics.py` 中的 `create_junction_dot()` 工厂函数标注

**画面炸成**：学生无法判断导线是否真正连接，电路逻辑混乱

---

#### C-D2：导线斜线转弯或斜向连接

- **禁止**：导线同时存在 X 和 Y 方向偏移（斜线）
- **必须**：导线必须横平竖直，使用工厂函数创建水平/垂直线段

**画面炸成**：不符合 IEC 标准，视觉混乱

---

#### C-D3：元件缺少 connection_points 属性

- **禁止**：手动创建的元件未添加 connection_points
- **必须**：始终使用 `scripts/physics_graphics.py` 中的工厂函数创建元件（create_resistor / create_battery 等），自动包含 connection_points

**画面炸成**：无法自动连接，导线端点需要手动计算，容易出错

---

#### C-D4：回路未闭合或存在悬空端点

- **禁止**：导线终点未连接任何元件，形成悬空
- **必须**：使用 `build_series_circuit()` 等工厂函数确保所有导线形成闭合回路

**画面炸成**：电路断路，电流无法流通

---

#### C-D5：电源正负极标识缺失

- **禁止**：电池符号缺少 "+" / "-" 标记
- **必须**：使用 `create_battery()` 工厂函数，自动包含极性标记

**画面炸成**：无法区分正负极，违反 IEC 60617-02 强制要求

## 字幕负向约束（Don't）

> **用途**：当字幕代码写成这样 → 画面炸成这样。Agent 必须避免以下任意一条。

### S-D1：单条字幕过长（单行超过20字）

### S-D4：字幕区 Y 超出边界

- **禁止**：字幕放在正区域，与主内容重叠
- **必须**：字幕固定在字幕区（Y ∈ [-3.85, -2.8]）

**画面炸成**：字幕与公式重叠，遮挡主内容

### S-D5：单行字幕字符超过 20

- **禁止**：单行字符过长，远端无法快速阅读
- **必须**：使用 `split_utterance()` 限制每行 ≤ 20 字符

**画面炸成**：字幕溢出屏幕边缘

## 代码质量

- [ ] 包含所有必需的 import（manim, manim_voiceover）。
- [ ] 类名 PascalCase，文件名合理。
- [ ] 头部包含依赖安装注释和配置字典。
- [ ] 渲染命令包含 --disable_caching。
