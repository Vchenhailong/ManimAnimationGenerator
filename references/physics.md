## 2. 物理符号转义映射表（语音生成用）

在生成物理教学内容的语音文本时，必须将以下物理符号和单位转换为自然语言。

### 2.1 物理量符号

| 符号        | 转义后       | 示例                                                                  |
| ----------- | ------------ | --------------------------------------------------------------------- |
| \vec{F}     | 力矢量 F     | \vec{F} = m\vec{a} → 力矢量 F 等于 m 乘以加速度矢量 a                 |
| \vec{G}     | 重力 G       | \vec{G} = mg → 重力 G 等于 m g                                        |
| \vec{f}     | 摩擦力 f     | \vec{f} = \mu N → 摩擦力 f 等于 mu N                                  |
| \vec{v}     | 速度 v       | \vec{v} = \frac{dx}{dt} → 速度 v 等于 d x 比 d t                      |
| \vec{a}     | 加速度 a     | \vec{a} = \frac{dv}{dt} → 加速度 a 等于 d v 比 d t                    |
| \vec{E}     | 电场强度 E   | \vec{E} = \frac{F}{q} → 电场强度 E 等于 F 除以 q                      |
| \vec{B}     | 磁感应强度 B | \vec{B} → 磁感应强度 B                                                |
| I           | 电流 I       | I = \frac{U}{R} → 电流 I 等于 U 除以 R                                |
| U           | 电压 U       | U = IR → 电压 U 等于 I R                                              |
| R           | 电阻 R       | R = \frac{U}{I} → 电阻 R 等于 U 除以 I                                |
| C           | 电容 C       | C = \frac{Q}{U} → 电容 C 等于 Q 除以 U                                |
| L           | 电感 L       | L → 电感 L                                                            |
| \Phi        | 磁通量       | \Phi = B \cdot S → 磁通量 Phi 等于 B 点乘 S                           |
| \varepsilon | 电动势       | \varepsilon = - \frac{d\Phi}{dt} → 电动势 epsilon 等于负 d Phi 比 d t |

# 第2部分：力学

## 4. 力矢量绘制规范

### 4.1 基本原则

| 要素     | 规范                                                 | 标准来源           |
| -------- | ---------------------------------------------------- | ------------------ |
| 箭头样式 | **实心三角箭头**（非空心），箭头长度约线长的 1/5     | GB/T 4460-2013 7.1 |
| 作用点   | 起点**必须**位于受力物体的重心或实际受力点           | GB/T 4460-2013 7.2 |
| 方向     | 与力的实际方向严格一致                               | GB/T 4460-2013 7.3 |
| 坐标系   | 数学坐标系（Y轴向上为正），通过 `axes.c2p(x,y)` 转换 | 本技能坐标系约束   |

> ⚠️ **禁止**：在代码中直接使用 `Arrow` 手动计算起终点坐标、硬编码颜色。
> 应始终使用 `create_force_arrow()` 工厂函数，其内部自动处理方向向量归一化、长度比例尺（0.5 单位/牛顿）和颜色映射。

### 4.2 受力点选择规范

**核心原则**：在绘制受力分析图时，必须以正确的力学分析结果和规范的工程制图方法为前提，并且所有力矢量的作用点必须准确落在受力对象的实际受力点上。

#### 基本分类

| 类别                | 定义                                                                               | 典型代表                       | 作用点确定方式                         |
| ------------------- | ---------------------------------------------------------------------------------- | ------------------------------ | -------------------------------------- |
| **场力 / 广义体力** | 作用于物体的每一个质点上，等效为作用于**重心（质心）**                             | 重力 G、浮力 F\_浮             | `object.get_center()`                  |
| **接触力**          | 仅在两个物体相互接触的表面产生，作用于**实际接触点或接触面中点**                   | 支持力 N、摩擦力 f、压力 F\_压 | `object.get_bottom()` / 接触面几何中点 |
| **约束力 / 外加力** | 通过绳、杆、弹簧等介质传递，或由外部施力者施加，**作用点取决于连接位置或施力位置** | 绳张力 T、推力 F*推、拉力 F*拉 | **由场景设计决定**                     |

#### 受力点速查表

| 力类型   | 正确作用点                                            | Manim 对应 API                                                |
| -------- | ----------------------------------------------------- | ------------------------------------------------------------- |
| G 重力   | 物体质心/几何中心（均匀物体 = 几何中心）              | `obj.get_center()`                                            |
| N 支持力 | 接触面中点（水平面→底部中心点；斜面→垂直接触点）      | `obj.get_bottom()` + surface_offset 或 contact_point 预计算值 |
| f 摩擦力 | 接触面上切向点（与 N 同一点，共点异向）               | 与 N 同一点                                                   |
| F 推力   | 施力接触点（手推箱子→侧面中心偏上；杠杆端→杆的末端）  | **由场景决定**                                                |
| T 张力   | 绳/杆连接点（悬挂物→物体顶部系绳点；斜拉→侧面挂钩点） | **由场景决定**                                                |

#### 关键规则

1. **G 和 N/f 必须共点于接触面的投影关系**
2. **摩擦力 f 与支持力 N 共点**
3. **F/T 类力的作用点是设计决策，必须在代码注释中说明理由**
4. **物体经旋转/平移变换后，静态快照坐标失效** — 必须使用动态查询

### 4.4 标注完整性规则

每个力矢量**必须包含两项标注**：

- **符号标注**：箭头末端外侧标注 LaTeX 符号（`\vec{G}`, `\vec{N}`, `\vec{f}` 等），字号 22
- **名称标注**（强制）：符号旁或下方标注中文名称（"重力"、"支持力"、"摩擦力 f"），字号 22

### 4.5 长度比例尺

- 统一比例尺：`scale = 0.5` 单位/牛顿（可按场景整体调整）
- 箭头长度 = 力的大小 × scale
- 同一场景中所有力矢量必须使用**相同比例尺**

# 第3部分：电磁学

## 7. 电场与磁场

### 7.1 电场线绘制

- 电场线从正电荷指向负电荷，密度与场强成正比，不相交。

### 7.2 磁感线绘制

- 闭合曲线，不相交，从 N 指向 S（外部）
- 疏密表示场强（密 → 强）
- 方向箭头沿磁感线，均匀分布（至少 3 个/线）
- 内部磁场：若展示剖面，用直线箭头从 S→N

### 7.3 垂直纸面方向符号

| 方向         | 符号 | Unicode | Manim 表示        |
| ------------ | ---- | ------- | ----------------- |
| 垂直纸面向外 | ⊙    | U+2299  | `Tex(r"\odot")`   |
| 垂直纸面向里 | ⊗    | U+2297  | `Tex(r"\otimes")` |

### 7.4 右手螺旋定则（安培定则）

- 分步演示：
  1. 显示导线/螺线管，标注电流方向（红色箭头）
  2. 伸出右手模型
  3. 高亮电流对应四指/拇指方向
  4. 渐显磁感线（按序从内向外）
- 对于螺线管：同时展示剖面图（内部箭头从 S->N）

### 7.5 左手定则（电动机）

- 使用辅助坐标系（x, y, z）标注三个方向：
  - 磁场 B：沿 x 或 y 方向（绿色）
  - 电流 I：沿另一水平方向（红色）
  - 受力 F：垂直纸面方向（蓝色），用 ⊙/⊗ 表示
- 分步高亮顺序：显示磁场线 → 显示电流方向 → 显示受力方向

### 7.6 右手定则（发电机 / 楞次定律）

- 切割磁感线场景：三要素同时高亮（磁场方向、运动方向、感应电流方向）
- 楞次定律动画时序：
  1. 显示线圈和磁铁
  2. 展示磁铁运动方向（箭头）
  3. 显示原磁场的磁感线（蓝色）
  4. 计算磁通量变化
  5. 显示感应电流产生的磁场方向（红色虚线磁感线）
  6. 最终显示感应电流方向
  7. 显示文字提示"增反减同"

# 第5部分：电路图元规范（IEC 60617 / GB/T 4728 / 人教版）

## 10. 元件符号标准

### 10.1 元件符号一览表

| 元件           | 符号规范                                   |
| -------------- | ------------------------------------------ |
| 电池           | 长线正极（+），短线负极（-），可标注电压值 |
| 电阻           | 矩形框，标注 R                             |
| 开关           | 断开时端点不接触，闭合时连接线             |
| 导线           | 直线或直角折线，禁止使用斜线转弯           |
| 节点（连接点） | 实心圆点 ●                                 |
| 电流表         | 圆圈内标 A                                 |
| 电压表         | 圆圈内标 V                                 |
| 电感线圈       | 螺旋线（若干半圆）                         |
| 电容           | 两条平行线                                 |

### 10.2 电池 (battery)

**Manim 表达方式**：`scripts/physics_graphics.create_battery()` 工厂函数

> ⚠️ **禁止**：在代码中直接使用 `VGroup(LongLine, ShortLine)` 手动组合。

| 参数     | 规范值/规则                        | 标准依据              |
| -------- | ---------------------------------- | --------------------- |
| 符号形状 | 长线 = 正极 (+)，短线 = 负极 (-)   | IEC 60617-02 S00001   |
| 长宽比   | 长:短 ≈ 2:1                        | IEC 60617-02 比例约定 |
| 引出方向 | 正极引出从长线端点，负极从短线端点 | GB/T 4728.02          |
| 标注格式 | `\mathcal{E}=xV` 或 `U=xV`         | ISO 80000-1           |
| **禁止** | 正负极标识缺失或颠倒               | IEC 60617 强制要求    |

### 10.3 电阻 (resistor)

**Manim 表达方式**：`scripts/physics_graphics.create_resistor()` 工厂函数

## 11. 元件连接点统一接口

### 11.0 核心术语：引脚（Port）与触点（Contact）

> **两者不是同一个东西**，但存在映射关系。

| 概念                   | 定义                                                    | 存在范围         | 代码体现                                                |
| ---------------------- | ------------------------------------------------------- | ---------------- | ------------------------------------------------------- |
| **引脚（Pin / Port）** | 元件对外暴露的**连接接口**，供外部 `create_wire()` 接入 | **所有元件**     | `.start_port`, `.end_port`, `.left_port`, `.right_port` |
| **触点（Contact）**    | 开关/继电器等机械式元件的**物理接触点**（圆点符号）     | **仅开关类元件** | `dot_a`(固定端), `dot_b`(活动端)                        |

#### 映射关系

```
┌─────────────────────────────────────────────────────────┐
│                    工厂函数输出                           │
│                                                         │
│  ┌──────────┐    .start_port    ┌──────────────────┐   │
│  │ 引脚/端口 │◄══ 映射指向 ════►│ 实际连接坐标      │   │
│  │ (Port)   │                  │ (供create_wire用)  │   │
│  └──────────┘                 └──────────────────┘   │
│                                                         │
│  【无触点元件】引脚 → 元件本体边缘                        │
│    电池: start_port → 短线(负极) / end_port → 长线(正极)  │
│    电阻: start_port → 左端 / end_port → 右端             │
│    电表: left_port → 圆周左 / right_port → 圆周右        │
│                                                         │
│  【有触点元件】引脚 = 触点坐标（直接重合）                │
│    开关: start_port = dot_a(固定触点)                    │
│          end_port   = dot_b(活动触点)                    │
│                                                         │
│  设计原则：工厂函数只输出纯元件本体+触点，零自带导线       │
│            所有连线由调用方用 create_wire() 独立绘制       │
└─────────────────────────────────────────────────────────┘
```

#### 关键约束

1. **AI 接线层必须使用引脚（Port）属性**（`.start_port/.end_port`），禁止直接访问触点（`dot_a/dot_b`）
2. **工厂函数禁止自带导线**：`create_wire()` 是唯一的导线来源
3. **引脚坐标 = 可连接点坐标**：`create_wire(前一元件.end_port, 后一元件.start_port)` 必须一次成功

### 11.1 获取连接点函数

## 13. 电路数理验证规则

> ⚠️ **强制**：生成电路图代码后必须执行以下验证。

### 13.1 验证 A：数理验证（推荐）

**可验证的电路属性**：

| 属性             | 验证方法                                                     | 判定标准                          |
| ---------------- | ------------------------------------------------------------ | --------------------------------- |
| **导线连通性**   | 从电源正极出发，遍历所有导线 end 与下一元件 start 的欧氏距离 | 距离 < 0.01 视为连通              |
| **节点等电势**   | 并联支路接入点的坐标一致性检查                               | 同一节点上所有导线起点/Y 坐标一致 |
| **元件串联关系** | 元件间导线长度 > 0                                           | 符合电路拓扑设计                  |
| **L 型连接识别** | 三条导线交汇点的坐标重合性验证（junction_dot 位置）          | 三条导线端点坐标距离 < 0.01       |

调用 `scripts/physics_graphics.py` 中的 `validate_circuit_topology()` 函数执行验证。

### 13.2 验证 B：规范校验（当无法数理验证时 fallback）

**手动检查清单（必须逐项确认）**：

- [ ] **电源正负极标识完整**：电池符号中 "+"/"-" 标记可见（IEC 60617-02 强制）
- [ ] **导线横平竖直走线**：所有导线段角度为 0°/90°/180°/270°（禁止斜线）
- [ ] **并联挂载三段式**：横出段→纵跨段→接入段（见 netlist.md §四.1）
- [ ] **T 型连接有节点圆点**：三条导线交汇点有节点圆点标注
- [ ] **节点圆点尺寸足够**：半径 = 0.04（技能默认值 0.04，GB/T 4728 标准最小要求）
- [ ] **导线交叉无连接处有跨线弧**：使用跨线弧处理
- [ ] **元件导线拆分正确**：任何的元件（如电阻/开关/电容）两端导线各自独立（禁止单线穿过）
- [ ] **并联支路完整**：每支路从主节点出发 → 经过元件 → 返回另一主节点
- [ ] **无悬空端点**：所有导线终点都连接到元件端点或节点
- [ ] **无零长度线段**：任何线段的起点 ≠ 终点
