## 2. 物理符号转义映射表（语音生成用）

在生成物理教学内容的语音文本时，必须将以下物理符号和单位转换为自然语言。

### 2.1 物理量符号

| 符号        | 转义后       | 示例                                                                  |
| ----------- | ------------ | --------------------------------------------------------------------- |
| \vec{F}     | 力矢量 F     | \vec{F} = m\vec{a} → 力矢量 F 等于 m 乘以加速度矢量 a                 |
| \vec{G}     | 重力 G       | \vec{G} = mg → 重力 G 等于 m g                                        |
| \vec{f}     | 摩擦力 f     | \vec{f} = \mu N → 摩擦力 f 等于 mu N                                  |
| \vec{v}     | 速度 v       | \vec{v} = \frac{dx}{dt} → 速度 v 等于 d x 比 d t                      |
| \vec{a}     | 加速度 a     | \vec{a} = \frac{dv}{dt} → 加速度 a 等于 d v 比 d t                    |
| \vec{E}     | 电场强度 E   | \vec{E} = \frac{F}{q} → 电场强度 E 等于 F 除以 q                      |
| \vec{B}     | 磁感应强度 B | \vec{B} → 磁感应强度 B                                                |
| I           | 电流 I       | I = \frac{U}{R} → 电流 I 等于 U 除以 R                                |
| U           | 电压 U       | U = IR → 电压 U 等于 I R                                              |
| R           | 电阻 R       | R = \frac{U}{I} → 电阻 R 等于 U 除以 I                                |
| C           | 电容 C       | C = \frac{Q}{U} → 电容 C 等于 Q 除以 U                                |
| L           | 电感 L       | L → 电感 L                                                            |
| \Phi        | 磁通量       | \Phi = B \cdot S → 磁通量 Phi 等于 B 点乘 S                           |
| \varepsilon | 电动势       | \varepsilon = - \frac{d\Phi}{dt} → 电动势 epsilon 等于负 d Phi 比 d t |

# 第2部分：力学

## 4. 力矢量绘制规范

### 4.1 基本原则

| 要素     | 规范                                                 | 标准来源           |
| -------- | ---------------------------------------------------- | ------------------ |
| 箭头样式 | **实心三角箭头**（非空心），箭头长度约线长的 1/5     | GB/T 4460-2013 7.1 |
| 作用点   | 起点**必须**位于受力物体的重心或实际受力点           | GB/T 4460-2013 7.2 |
| 方向     | 与力的实际方向严格一致                               | GB/T 4460-2013 7.3 |
| 坐标系   | 数学坐标系（Y轴向上为正），通过 `axes.c2p(x,y)` 转换 | 本技能坐标系约束   |

> ⚠️ **禁止**：在代码中直接使用 `Arrow` 手动计算起终点坐标、硬编码颜色。
> 应始终使用 `create_force_arrow()` 工厂函数，其内部自动处理方向向量归一化、长度比例尺（0.5 单位/牛顿）和颜色映射。

### 4.2 受力点选择规范

**核心原则**：在绘制受力分析图时，必须以正确的力学分析结果和规范的工程制图方法为前提，并且所有力矢量的作用点必须准确落在受力对象的实际受力点上。

#### 基本分类

| 类别                | 定义                                                                               | 典型代表                       | 作用点确定方式                         |
| ------------------- | ---------------------------------------------------------------------------------- | ------------------------------ | -------------------------------------- |
| **场力 / 广义体力** | 作用于物体的每一个质点上，等效为作用于**重心（质心）**                             | 重力 G、浮力 F\_浮             | `object.get_center()`                  |
| **接触力**          | 仅在两个物体相互接触的表面产生，作用于**实际接触点或接触面中点**                   | 支持力 N、摩擦力 f、压力 F\_压 | `object.get_bottom()` / 接触面几何中点 |
| **约束力 / 外加力** | 通过绳、杆、弹簧等介质传递，或由外部施力者施加，**作用点取决于连接位置或施力位置** | 绳张力 T、推力 F*推、拉力 F*拉 | **由场景设计决定**                     |

#### 受力点速查表

| 力类型   | 正确作用点                                            | Manim 对应 API                                                |
| -------- | ----------------------------------------------------- | ------------------------------------------------------------- |
| G 重力   | 物体质心/几何中心（均匀物体 = 几何中心）              | `obj.get_center()`                                            |
| N 支持力 | 接触面中点（水平面→底部中心点；斜面→垂直接触点）      | `obj.get_bottom()` + surface_offset 或 contact_point 预计算值 |
| f 摩擦力 | 接触面上切向点（与 N 同一点，共点异向）               | 与 N 同一点                                                   |
| F 推力   | 施力接触点（手推箱子→侧面中心偏上；杠杆端→杆的末端）  | **由场景决定**                                                |
| T 张力   | 绳/杆连接点（悬挂物→物体顶部系绳点；斜拉→侧面挂钩点） | **由场景决定**                                                |

#### 关键规则

1. **G 和 N/f 必须共点于接触面的投影关系**
2. **摩擦力 f 与支持力 N 共点**
3. **F/T 类力的作用点是设计决策，必须在代码注释中说明理由**
4. **物体经旋转/平移变换后，静态快照坐标失效** — 必须使用动态查询

### 4.4 标注完整性规则

每个力矢量**必须包含两项标注**：

- **符号标注**：箭头末端外侧标注 LaTeX 符号（`\vec{G}`, `\vec{N}`, `\vec{f}` 等），字号 22
- **名称标注**（强制）：符号旁或下方标注中文名称（"重力"、"支持力"、"摩擦力 f"），字号 22

### 4.5 长度比例尺

- 统一比例尺：`scale = 0.5` 单位/牛顿（可按场景整体调整）
- 箭头长度 = 力的大小 × scale
- 同一场景中所有力矢量必须使用**相同比例尺**

# 第3部分：电磁学

## 7. 电场与磁场

### 7.1 电场线绘制

- 电场线从正电荷指向负电荷，密度与场强成正比，不相交。

### 7.2 磁感线绘制

- 闭合曲线，不相交，从 N 指向 S（外部）
- 疏密表示场强（密 → 强）
- 方向箭头沿磁感线，均匀分布（至少 3 个/线）
- 内部磁场：若展示剖面，用直线箭头从 S→N

### 7.3 垂直纸面方向符号

| 方向         | 符号 | Unicode | Manim 表示        |
| ------------ | ---- | ------- | ----------------- |
| 垂直纸面向外 | ⊙    | U+2299  | `Tex(r"\odot")`   |
| 垂直纸面向里 | ⊗    | U+2297  | `Tex(r"\otimes")` |

### 7.4 右手螺旋定则（安培定则）

- 分步演示：
  1. 显示导线/螺线管，标注电流方向（红色箭头）
  2. 伸出右手模型
  3. 高亮电流对应四指/拇指方向
  4. 渐显磁感线（按序从内向外）
- 对于螺线管：同时展示剖面图（内部箭头从 S->N）

### 7.5 左手定则（电动机）

- 使用辅助坐标系（x, y, z）标注三个方向：
  - 磁场 B：沿 x 或 y 方向（绿色）
  - 电流 I：沿另一水平方向（红色）
  - 受力 F：垂直纸面方向（蓝色），用 ⊙/⊗ 表示
- 分步高亮顺序：显示磁场线 → 显示电流方向 → 显示受力方向

### 7.6 右手定则（发电机 / 楞次定律）

- 切割磁感线场景：三要素同时高亮（磁场方向、运动方向、感应电流方向）
- 楞次定律动画时序：
  1. 显示线圈和磁铁
  2. 展示磁铁运动方向（箭头）
  3. 显示原磁场的磁感线（蓝色）
  4. 计算磁通量变化
  5. 显示感应电流产生的磁场方向（红色虚线磁感线）
  6. 最终显示感应电流方向
  7. 显示文字提示"增反减同"

## 8. 电路绘制核心原则

> ⚠️ **最重要的原则**：电路图绘制必须遵循"**先获取坐标，再遵循规范**"的流程。

### 8.1 SVG 矢量图优先方案（强烈推荐）

> ⚠️ **重要声明**：由于 AI 在代码层面难以精确绘制复杂的电路图拓扑（涉及大量坐标计算、旋转变换、多段导线拼接），**强烈建议优先使用 SVG 矢量图方案**。

**为什么优先使用 SVG？**

| 方案           | 优点                                         | 缺点                                            |
| -------------- | -------------------------------------------- | ----------------------------------------------- |
| **SVG 矢量图** | 精确控制拓扑、所见即所得、易于导入专业电路图 | 需要外部工具或预制素材                          |
| Manim 代码绘制 | 可动画化、程序化生成                         | AI 难以精确控制坐标，容易出现导线断路、位置错位 |

**SVG 方案使用方式**：

```python
from manim import SVGMobject, config

# 方式 1：导入外部 SVG 文件
circuit_svg = SVGMobject("assets/circuits/mixed_circuit.svg")
self.add(circuit_svg)

# 方式 2：导入 PNG 图片（需要指定文件路径）
# SVG 文件应存储在 assets/ 目录下
# 每个电路图应为独立的 SVG 文件
```

**SVG 文件管理规范**：

| 规范     | 说明                                                     |
| -------- | -------------------------------------------------------- |
| 存放位置 | `assets/circuits/` 目录下                                |
| 文件命名 | `{电路类型}_{编号}.svg`，如 `mixed_circuit_01.svg`       |
| 尺寸规范 | 建议 800×600 像素或更高，保持比例                        |
| 颜色规范 | 使用规范指定的颜色（白色导线 #FFFFFF，元件轮廓 #000000） |

**何时使用 Manim 代码绘制 vs SVG？**

| 场景                               | 推荐方案                     |
| ---------------------------------- | ---------------------------- |
| 静态电路图（教材插图）             | SVG 矢量图                   |
| 需要动画演示（电流流动、开关切换） | Manim 代码绘制               |
| 简单串联电路                       | Manim 代码绘制（调试成本低） |
| 复杂混联电路、多层嵌套             | SVG 矢量图                   |
| 调试超过 3 次仍有问题              | 改用 SVG 方案                |

### 8.2 坐标优先原则（强制）

**绘制电路前，必须先完成以下步骤：**

```
┌─────────────────────────────────────────────────────────────────────┐
│                        电路绘制工作流程                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  步骤 1：规划整体布局                                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  确定电路区域尺寸、主回路走向（顺时针/逆时针）                  │   │
│  │  规划元件间距、节点位置                                       │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                           ↓                                        │
│  步骤 2：定义坐标常量                                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  # 示例代码                                                   │   │
│  │  r1_y  = 2.0     # R₁ 支路 Y 坐标                          │   │
│  │  r2_y  = 1.0     # R₂ 支路 Y 坐标                          │   │
│  │  main_y = 0.0     # 主回路 Y 坐标                           │   │
│  │  junc_a_x = -1.5  # 并联节点A X 坐标                        │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                           ↓                                        │
│  步骤 3：创建元件并获取连接点                                       │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  resistor1 = create_resistor(start, end, label)             │   │
│  │  # 通过 connection_points 获取元件端点坐标                    │   │
│  │  left_pt  = resistor1.connection_points["left"]             │   │
│  │  right_pt = resistor1.connection_points["right"]            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                           ↓                                        │
│  步骤 4：根据端点坐标绘制导线                                       │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  # 导线必须从元件端点出发，连接到下一元件端点                  │   │
│  │  wire1 = create_wire(left_pt, next_component_left)          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                           ↓                                        │
│  步骤 5：添加节点圆点和标注                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 8.3 元件自带导线的正确处理

> ⚠️ **关键理解**：`create_*` 工厂函数创建的元件通常自带短引出导线（wire_left/wire_right），这些引线仅用于标识连接位置。

**错误做法**：

```python
# ❌ 错误：直接创建元件，不考虑自带导线
resistor = create_resistor(start=[-1, 1, 0], end=[0, 1, 0])
# 然后期望导线能自动连接到电阻两端
```

**正确做法**：

```python
# ✅ 正确：获取元件连接点坐标
resistor = create_resistor(start=[-1, 1, 0], end=[0, 1, 0])
left_pt = resistor.connection_points["left"]   # 导线左端点
right_pt = resistor.connection_points["right"]  # 导线右端点

# 根据连接点坐标创建外接导线
wire_left = create_wire([prev_component_x, 1, 0], left_pt)
wire_right = create_wire(right_pt, [next_component_x, 1, 0])
```

### 8.4 常见错误及避免方法

| 错误类型               | 后果                       | 避免方法                                       |
| ---------------------- | -------------------------- | ---------------------------------------------- |
| 不获取元件坐标直接绘制 | 导线与元件位置不匹配，断路 | 先调用 factory 函数获取 `connection_points`    |
| 导线斜线转弯           | 违反"横平竖直"规范         | 使用多段折线：水平+垂直交替                    |
| 忘记添加节点圆点       | T 型连接不规范             | 在导线交汇处使用 `create_junction_dot()`       |
| 电压表/电流表连接错误  | 测量原理错误               | 电流表串联、电压表并联，根据连接点坐标精确绘制 |

## 12. 完整回路组装规范

### 12.1 核心原则

> 电路图以 `VGroup` 为单位组装为一个完整的 MObject。

### 12.2 组装方式对比

| 方式                      | 适用场景             | 说明                                       |
| ------------------------- | -------------------- | ------------------------------------------ |
| **手动组装（推荐）**      | 复杂电路、串并联混合 | 显式创建每个元件、导线和节点，完全控制布局 |
| `build_series_circuit()`  | 简单串联回路         | 自动生成矩形闭合回路，元件沿顶边水平排列   |
| `build_parallel_branch()` | 简单并联分支         | 在主回路上添加并联分支                     |

> ⚠️ **重要说明**：`build_series_circuit()` 和 `build_parallel_branch()` 存在设计局限：
>
> - 元件必须是连续的段（start/end）
> - 不支持带分离引出导线的元件（如开关）
> - 对于复杂电路，**推荐手动组装**以确保正确连接

### 12.3 手动组装流程

1. **创建元件**：使用 start/end 或 center 参数调用工厂函数
2. **获取连接点**：从元件的 `connection_points` 获取端点坐标
3. **创建导线**：使用 `create_wire()` 连接各元件，导线必须横平竖直
4. **创建节点圆点**：T 型连接必须使用 `create_junction_dot()` 标注
5. **合并为 VGroup**：将所有元件、导线、节点合并

### 12.4 导线走线规范

| 规则                                               | 强制等级 | 标准来源           |
| -------------------------------------------------- | -------- | ------------------ |
| 外接连接导线必须为**直角折线**（水平和垂直段交替） | MUST     | IEC 60617 通用规则 |
| 外接连接导线**禁止**斜线转弯                       | MUST NOT | 人教版教材         |
| 并联节点使用**实心圆点** 标注                      | MUST     | GB/T 4728          |
| 导线交叉但不连通处画**跨线弧**（小圆弧）           | MUST     | GB/T 4728          |
| 导线交叉且连通处画**结点**（实心圆点）             | MUST     | GB/T 4728          |
| 导线颜色统一 WHITE 或 LIGHT_GRAY, width=2          | SHOULD   | 人教版教材         |

### 12.5 回路路径追踪验证

**目的**：确保电路图满足"从电源正极出发，能遍历所有元件并回到电源负极"的物理连通性要求。

**核心原则**：电路图必须包含至少一个**闭合回路**，即从电源正极出发，沿着导线能够依次经过所有应该导通的元件，最终回到电源负极。

**回路路径追踪步骤**：

1. **标识电源端点**：找到电池/电源元件，标记正极 (+) 和负极 (-)
2. **选择起点**：从电源正极端子开始
3. **追踪路径**：沿着导线移动，记录经过的每个元件和节点
4. **检查连通性**：
   - 每段导线必须连接到某个元件端点或另一个导线段
   - 元件的每个连接点必须连接到导线
5. **验证终点**：路径必须能回到电源负极

## 13. 电路数理验证规则

> ⚠️ **强制**：生成电路图代码后必须执行以下验证。

### 13.1 验证 A：数理验证（推荐）

**可验证的电路属性**：

| 属性             | 验证方法                                                     | 判定标准                          |
| ---------------- | ------------------------------------------------------------ | --------------------------------- |
| **导线连通性**   | 从电源正极出发，遍历所有导线 end 与下一元件 start 的欧氏距离 | 距离 < 0.01 视为连通              |
| **节点等电势**   | 并联支路接入点的坐标一致性检查                               | 同一节点上所有导线起点/Y 坐标一致 |
| **元件串联关系** | 元件间导线长度 = 0（直接连接）或 > 0（有导线间隙）           | 符合电路拓扑设计                  |
| **T 型连接识别** | 三条导线交汇点的坐标重合性验证（junction_dot 位置）          | 三条导线端点坐标距离 < 0.01       |

调用 `scripts/physics_graphics.py` 中的 `validate_circuit_topology()` 函数执行验证。

### 13.2 验证 B：规范校验（当无法数理验证时 fallback）

**手动检查清单（必须逐项确认）**：

- [ ] **电源正负极标识完整**：电池符号中 "+"/"-" 标记可见（IEC 60617-02 强制）
- [ ] **导线横平竖直走线**：所有导线段角度为 0°/90°/180°/270°（允许多段）
- [ ] **T 型连接有节点圆点**：三条导线交汇点有节点圆点标注
- [ ] **节点圆点尺寸足够**：半径 = 0.04（技能默认值 0.04，GB/T 4728 标准最小要求）
- [ ] **导线交叉无连接处有跨线弧**：使用跨线弧处理
- [ ] **元件导线拆分正确**：电阻/开关/电容两端导线各自独立（禁止单线穿过）
- [ ] **并联支路完整**：每支路从主节点出发 → 经过元件 → 返回另一主节点
- [ ] **无悬空端点**：所有导线终点都连接到元件端点或节点
- [ ] **无零长度线段**：任何线段的起点 ≠ 终点

### 13.3 验证 C：程序化拓扑校验（推荐）

调用 `scripts/physics_graphics.py` 中的 `validate_circuit_topology()` 自动检查。

**返回值**：

- `is_valid=True`：拓扑通过校验
- `is_valid=False`：返回 `errors` 列表，每项为具体错误描述
