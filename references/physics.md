# 物理学科专属规范

## 1. 力学

- 受力分析：力矢量从作用点出发，长度与大小成正比。
- 颜色区分，色值差大于45度：重力 #66FFAA（绿），支持力/弹力 #66DDFF（蓝），摩擦力 #FF6666（红），外力 #FFDD66（黄）。
- 标签：\vec{F}\_N 等，字号 28，置于矢量中点外侧。
- 运动轨迹：虚线 + 箭头，速度矢量与轨迹相切，加速度指向凹侧。
- 约束：轻绳/轻杆细实线，光滑接触面虚线法线，弹簧锯齿线。

## 2. 电磁学

- 电场线从正电荷指向负电荷，密度与场强成正比，不相交。
- 磁感线闭合，从 N 到 S（外部），内部从 S 到 N。
- 通电直导线磁场为同心圆，方向用 ⊙（出）⊗（入）标注。
- 电路：元件符号标准化，导线直角走线，节点实心圆点。

## 3. 波动与光学

- 波形图：横轴位置/时间，纵轴位移；横波为正弦曲线，标注 A、λ。
- 光线图：带箭头直线，法线虚线，入射/反射/折射角用弧线标注。
- 透镜：双弧线（凸），双凹陷（凹），标注 F、f。

## 6. 物理符号转义映射表（语音生成用）

在生成物理教学内容的语音文本时，必须将以下物理符号和单位转换为自然语言。

### 6.1 物理量符号

| 符号        | 转义后       | 示例                                                                   |
| ----------- | ------------ | ---------------------------------------------------------------------- |
| \vec{F}     | 力矢量 F     | \vec{F} = m\vec{a} -> 力矢量 F 等于 m 乘以加速度矢量 a                 |
| \vec{G}     | 重力 G       | \vec{G} = mg -> 重力 G 等于 m g                                        |
| \vec{f}     | 摩擦力 f     | \vec{f} = \mu N -> 摩擦力 f 等于 mu N                                  |
| \vec{v}     | 速度 v       | \vec{v} = \frac{dx}{dt} -> 速度 v 等于 d x 比 d t                      |
| \vec{a}     | 加速度 a     | \vec{a} = \frac{dv}{dt} -> 加速度 a 等于 d v 比 d t                    |
| \vec{E}     | 电场强度 E   | \vec{E} = \frac{F}{q} -> 电场强度 E 等于 F 除以 q                      |
| \vec{B}     | 磁感应强度 B | \vec{B} -> 磁感应强度 B                                                |
| I           | 电流 I       | I = \frac{U}{R} -> 电流 I 等于 U 除以 R                                |
| U           | 电压 U       | U = IR -> 电压 U 等于 I R                                              |
| R           | 电阻 R       | R = \frac{U}{I} -> 电阻 R 等于 U 除以 I                                |
| C           | 电容 C       | C = \frac{Q}{U} -> 电容 C 等于 Q 除以 U                                |
| L           | 电感 L       | L -> 电感 L                                                            |
| \Phi        | 磁通量       | \Phi = B \cdot S -> 磁通量 Phi 等于 B 点乘 S                           |
| \varepsilon | 电动势       | \varepsilon = - \frac{d\Phi}{dt} -> 电动势 epsilon 等于负 d Phi 比 d t |

### 8.1 元件符号

| 元件           | 符号规范                                   |
| -------------- | ------------------------------------------ |
| 电池           | 长线正极（+），短线负极（-），可标注电压值 |
| 电阻           | 锯齿线（曲折线），或矩形框，标注 R         |
| 开关           | 断开时端点不接触，闭合时连接线             |

| 节点（连接点） | 实心圆点 ●                                 |
| 电流表         | 圆圈内标 A                                 |
| 电压表         | 圆圈内标 V                                 |
| 电感线圈       | 螺旋线（若干半圆）                         |
| 电容           | 两条平行线                                 |

### 8.2 电流方向标注

- 正电荷移动方向（传统方向）：箭头沿导线外侧，标注 I
- 电子流动方向（可选）：虚线箭头 + e⁻ 标记
- 分支电流：标注 I₁, I₂ 等

### 8.3 电路可通性判断可视化

- 通路：绿色高亮路径，箭头动画展示电流流动
- 断路：在断点处显示红色叉号 ✗，电流路径变灰
- 短路：用黄色闪烁高亮，并显示警告符号 ⚠

### 8.4 电路图数理验证规则（强制）

> **生成电路图代码后必须执行以下验证，确保满足任意一项：**

#### 验证 A：数理验证（推荐）

**可验证的电路属性**：

| 属性             | 验证方法                                                     | 判定标准                          |
| ---------------- | ------------------------------------------------------------ | --------------------------------- |
| **导线连通性**   | 从电源正极出发，遍历所有导线 end 与下一元件 start 的欧氏距离 | 距离 < 0.01 视为连通              |
| **节点等电势**   | 并联支路接入点的坐标一致性检查                               | 同一节点上所有导线起点/Y 坐标一致 |
| **元件串联关系** | 元件间导线长度 = 0（直接连接）或 > 0（有导线间隙）           | 符合电路拓扑设计                  |
| **T 型连接识别** | 三条导线交汇点的坐标重合性验证（junction_dot 位置）          | 三条导线端点坐标距离 < 0.01       |

**数理验证代码示例**：

```python
def validate_circuit_topology(circuit: VGroup) -> Tuple[bool, List[str]]:
    """验证电路拓扑完整性（生成时自动执行）"""
    errors = []

    # 1. 检查所有元件的 connection_points 是否存在
    for component in circuit:
        if not hasattr(component, 'connection_points'):
            errors.append(f"元件 {component} 缺少 connection_points 属性")

    # 2. 检查导线连通性（从电池正极出发追溯）
    battery = next((c for c in circuit if getattr(c, 'component_type', None) == 'battery'), None)
    if battery:
        # 从正极 start 出发，递归检查所有导线和元件
        visited = set()
        def trace_connectivity(pos: np.ndarray) -> bool:
            # 检查是否有导线或元件端点在此位置
            for comp in circuit:
                if hasattr(comp, 'connection_points'):
                    for name, pt in comp.connection_points.items():
                        if np.linalg.norm(pt - pos) < 0.01 and id(comp) not in visited:
                            visited.add(id(comp))
                            # 递归追溯该元件另一端
                            if hasattr(comp, 'wire_right') or hasattr(comp, 'wire_left'):
                                return trace_connectivity(comp.connection_points['end'])
            return True
        trace_connectivity(battery.connection_points['start'])

    return len(errors) == 0, errors
```

#### 验证 B：规范校验（当无法数理验证时 fallback）

**手动检查清单（必须逐项确认）**：

- [ ] **电源正负极标识完整**：电池符号中 "+"/"-" 标记可见（IEC 60617-02 强制）
- [ ] **导线横平竖直走线**：所有导线段角度为 0°/90°/180°/270°（允许多段）
- [ ] **T 型连接有节点圆点**：三条导线交汇点有 `create_junction_dot()` 标注
- [ ] **导线交叉无连接处有跨线弧**：使用 `create_arc_bridge()` 处理
- [ ] **元件导线拆分正确**：电阻/开关/电容两端导线各自独立（禁止单线穿过）
- [ ] **并联支路完整**：每支路从主节点出发 → 经过元件 → 返回另一主节点
- [ ] **无悬空端点**：所有导线终点都连接到元件端点或节点
- [ ] **无零长度线段**：任何 `Line(start, end)` 的 `start != end`

## 9. 磁场方向标注规范

### 9.1 磁感线绘制

- 闭合曲线，不相交，从 N 指向 S（外部）
- 疏密表示场强（密 -> 强）
- 方向箭头沿磁感线，均匀分布（至少 3 个/线）
- 内部磁场：若展示剖面，用直线箭头从 S 指向 N

### 9.2 垂直纸面方向符号（强制）

| 方向         | 符号 | Unicode | Manim 表示      |
| ------------ | ---- | ------- | --------------- |
| 垂直纸面向外 | ⊙    | U+2299  | Tex(r"\odot")   |
| 垂直纸面向里 | ⊗    | U+2297  | Tex(r"\otimes") |

### 9.3 右手螺旋定则（安培定则）动画要求

- 分步演示：
  1. 显示导线/螺线管，标注电流方向（红色箭头）
  2. 伸出右手模型（可用 3D 或 2D 投影手型图标）
  3. 高亮电流对应四指/拇指方向
  4. 渐显磁感线（按序从内向外）
- 对于螺线管：同时展示剖面图（内部箭头从 S->N）

## 10. 左手定则（电动机）动画要求

### 10.1 三维垂直关系表示

- 使用辅助坐标系（x, y, z）标注三个方向：
  - 磁场 B：沿 x 或 y 方向（绿色）
  - 电流 I：沿另一水平方向（红色）
  - 受力 F：垂直纸面方向（蓝色），用 ⊙/⊗ 表示
- 或采用分视角 2D 图：分别展示主视图（B 和 I）、侧视图（F 方向）

### 10.2 分步高亮顺序

1. 显示磁场线（绿色渐变），标注 B
2. 显示电流方向（红色箭头），标注 I
3. 显示受力方向（蓝色箭头），标注 F
4. 用左手模型图标辅助，依次高亮：磁感线穿手心 -> 四指指电流 -> 拇指指受力

## 11. 右手定则（发电机 / 楞次定律）动画要求

### 11.1 切割磁感线场景

- 展示导体棒在 U 形导轨上滑动
- 三要素同时高亮：
  - 磁场方向（绿色，垂直纸面或上下）
  - 导体运动方向（黄色箭头）
  - 感应电流方向（红色箭头沿回路）
- 辅助使用右手模型图标：拇指运动，四指感应电流

### 11.2 楞次定律（磁铁插入/拔出线圈）

- 动画时序：
  1. 显示线圈和磁铁
  2. 展示磁铁运动方向（箭头）
  3. 显示原磁场的磁感线（蓝色）
  4. 计算磁通量变化（增大或减小，用仪表盘动画模拟）
  5. 显示感应电流产生的磁场方向（红色虚线磁感线）
  6. 最终显示感应电流方向（沿线圈箭头）
- 同时显示文字提示增反减同

## 12. 电路与磁场混合场景（电动机、发电机整机演示）

### 12.1 直流电动机

#### 15.2.1 力矢量箭头 (force_arrow)

**标准依据**：GB/T 4460-2013 第 7 章（力的表示法）/ 人教版高中物理必修一插图规范

**Manim 表达方式**：`scripts/physics_graphics.create_force_arrow()` 工厂函数


### 15.1.1 受力点选择规范（强制）

**核心原则**：在绘制受力分析图时，必须以正确的力学分析结果和规范的工程制图方法为前提，并且所有力矢量的作用点必须准确落在受力对象的实际受力点上。

#### 基本分类

物理中的力按其作用点的确定性可分为两类：

| 类别                | 定义                                                                               | 典型代表                       | 作用点确定方式                         |
| ------------------- | ---------------------------------------------------------------------------------- | ------------------------------ | -------------------------------------- |
| **场力 / 广义体力** | 作用于物体的每一个质点上，等效为作用于**重心（质心）**                             | 重力 G、浮力 F\_浮             | `object.get_center()`                  |
| **接触力**          | 仅在两个物体相互接触的表面产生，作用于**实际接触点或接触面中点**                   | 支持力 N、摩擦力 f、压力 F\_压 | `object.get_bottom()` / 接触面几何中点 |
| **约束力 / 外加力** | 通过绳、杆、弹簧等介质传递，或由外部施力者施加，**作用点取决于连接位置或施力位置** | 绳张力 T、推力 F*推、拉力 F*拉 | **由场景设计决定**，需明确标注         |

#### 各类力的作用点选取细则

```
┌─────────────────────────────────────────────────────────────────────┐
│                    受力点速查表                                      │
├──────────┬──────────────────────┬───────────────────────────────────┤
│ 力类型   │ 正确作用点            │ Manim 对应 API                    │
├──────────┼──────────────────────┼───────────────────────────────────┤
│ G 重力   │ 物体质心/几何中心     │ obj.get_center()                 │
│          │ （均匀物体 = 几何中心） │                                  │
├──────────┼──────────────────────┼───────────────────────────────────┤
│ N 支持力 │ 接触面中点            │ obj.get_bottom() + surface_offset │
│          │ （水平面→底部中心点）  │ 或 contact_point 预计算值         │
│          │ （斜面→垂直接触点）    │                                  │
├──────────┼──────────────────────┼───────────────────────────────────┤
│ f 摩擦力 │ 接触面上切向点        │ 与 N 同一点（共点异向）           │
│          │ （与支持力共点但方向沿切线）│                               │
├──────────┼──────────────────────┼───────────────────────────────────┤
│ F 推力   │ 施力接触点            │ **由场景决定**：                  │
│          │                      │ 手推箱子 → 箱子侧面中心偏上        │
│          │                      │ 杠杆端 → 杆的末端                  │
├──────────┼──────────────────────┼───────────────────────────────────┤
│ T 张力   │ 绳/杆连接点           │ **由场景决定**：                  │
│          │                      │ 悬挂物 → 物体顶部系绳点            │
│          │                      │ 斜拉 → 物体侧面挂钩点              │
└──────────┴──────────────────────┴───────────────────────────────────┘
```

#### 关键规则

1. **G 和 N/f 必须共点于接触面的投影关系**
   - 水平面上的物体：G 从质心向下，N 从底面向上，两者在同一条竖直线上
   - 斜面上的物体：G 仍从质心竖直向下，N 从接触面垂直斜面向上（不与 G 共线）
   - **错误画法**：N 的起点不在接触面上，或 G 的起点偏离质心

2. **摩擦力 f 与支持力 N 共点**
   - f 和 N 同为接触力，作用点相同，方向互相垂直（f ∥ 接触面，N ⊥ 接触面）
   - 在图中应表现为从同一点出发的两个箭头

3. **F/T 类力的作用点是设计决策，必须在代码注释中说明理由**

   ```python
   # 推力 F：手推车把中点（模拟人手握持位置）
   push_force = create_force_arrow(
       origin=car.named_parts["body"].get_top() + UP * 0.1,  # 车把位置
       direction_vector=[0.7, 0.3, 0],  # 斜向上推
       ...
   )
   ```

4. **物体经旋转/平移变换后，静态快照坐标失效**
   - 工厂函数返回的 `connection_points` 是创建时刻的坐标快照
   - 若后续对物体执行了 `.rotate()` / `.shift()`，必须使用动态查询：
     - `obj.get_center()` — 当前质心
     - `obj.get_bottom()` — 当前底部
     - `obj.named_parts["body"].get_right()` — 当前命名部件的位置

#### 常见错误对照

| 错误现象                         | 根因                                | 正确做法                                |
| -------------------------------- | ----------------------------------- | --------------------------------------- |
| 重力箭头从物体外部出发           | 使用了过期的 connection_points 坐标 | 用 `obj.get_center()` 动态获取当前质心  |
| 支持力不从接触面出发             | origin 直接用 center 而非 bottom    | 水平面用 `get_bottom()`；斜面需计算垂足 |
| 摩擦力与支持力不共点             | 分别传了不同的 origin 参数          | N 和 f 应使用完全相同的 origin 坐标     |
| 推力作用点不合理（如在车轮中心） | 未考虑实际的物理施力场景            | 根据场景描述确定合理的施力接触点        |

**适配判定流程**：

```
1. 确认当前场景背景色（Scene.background_color 或默认 BLACK）
2. 对比每个力矢量的基准色与背景色的 HSL 差异：
   - 若 ΔL > 30%（明度差足够大）→ 直接使用基准色
   - 若 ΔH < 25° 且 ΔS < 20%（色相近且饱和度都高）→ 必须替换
3. 从上表查找对应背景+力的替代组合
4. 若表中无对应条目 → 手动选择：保持原色相 H 不变，
   将 L 向远离背景的方向调整至少 20%
```

**标注完整性规则（强制）**：

每个力矢量**必须包含两项标注**：

```
        ↑ N
        │ 支持力
   ─────┼───── 物体
        │
        ↓ G  重力
```

- **符号标注**：箭头末端外侧标注 LaTeX 符号（`\vec{G}`, `\vec{N}`, `\vec{f}` 等），字号 28
- **名称标注**（强制）：符号旁或下方标注中文名称（"重力"、"支持力"、"摩擦力 f"），字号略小(24)

**长度比例尺**：

- 统一比例尺：`scale = 0.5` 单位/牛顿（可按场景整体调整）
- 箭头长度 = 力的大小 × scale
- 同一场景中所有力矢量必须使用**相同比例尺**
- 多个力共存时，箭头长度**大致反映力的大小比例**（定性示意即可）

**导线走线规范** — 对齐 IEC 60617 / GB/T 4728 / 人教版：

| 规则                                        | 强制等级 | 标准来源           |
| ------------------------------------------- | -------- | ------------------ |
| 优先**直角折线**（水平和垂直段交替）        | MUST     | IEC 60617 通用规则 |
| **禁止**斜线转弯（除非题目明确斜向布线）    | MUST NOT | 人教版教材         |

| 导线交叉但不连通处画**跨线弧**（小圆弧）    | MUST     | GB/T 4728          |
| 导线交叉且连通处画**结点**（实心圆点）      | MUST     | GB/T 4728          |
| 导线颜色统一 WHITE 或 LIGHT_GRAY, width=2   | SHOULD   | 人教版教材         |

**电路拓扑完整性检查清单（生成时逐项确认）**：

- [ ] **闭合回路**：电源正极有导线引出 → 经过所有串联/并联元件 → 回到电源负极
- [ ] **分支完整**：每个并联分支从主回路节点出发 → 经过分支内所有元件 → 返回另一主回路节点
- [ ] **断开分支仍画导线**：`switch(closed=false)` 的分支导线仍完整绘制；断开由 switch 自身表达
- [ ] **无悬空端点**：导线终点必须有连接对象
- [ ] **无零长度线段**：任何 `Line` 的起点 ≠ 终点

```python
def build_buoyancy_scene(object_mass, object_volume, liquid_density):
    """构建浮力实验完整场景（遵循阿基米德原理可视化规范）"""
    container = create_container(points=[(-1, 0), (1, 0), (0.8, 2.5), (-0.8, 2.5)])
    displaced_volume = calculate_submerged_volume(object_volume, liquid_density, ...)
    liquid = create_liquid(container, fill_height=displaced_volume / container_area)

    # 力矢量（遵循 15.2.1 颜色映射表）
    gravity = create_force_arrow(
        origin=obj.connection_points["center"],
        direction_vector=[0, -1, 0],
        magnitude=object_mass * 9.8,
        label=r"\vec{G}",
    )
    buoyancy = create_force_arrow(
        origin=obj.connection_points["center"],
        direction_vector=[0, 1, 0],
        magnitude=liquid_density * displaced_volume * 9.8,
        label=r"\vec{F}_{\text{浮}}",
    )
    return VGroup(container, liquid, obj, gravity, buoyancy)
```

> ⚠️ **禁止**：`VGroup` 中各子元件必须独立使用工厂函数创建后组装，禁止混用手工 `Polygon` / `Arrow` / `Circle`。

## 16. 物理绘图负向约束（Don't）

> **用途**：当物理图元代码写成这样 → 画面炸成这样。Agent 必须避免以下任意一条。
> 对应 SKILL.md 中的 [负向约束速查索引](../SKILL.md#负向约束速查索引dont-quick-reference)。

### P-D1：力矢量颜色不符合力-色映射表

```python
# ❌ DON'T：使用不符合规范的配色（与背景/其他力混淆）
bad_arrow = Arrow(color=ORANGE)    # 橙色不在五色映射中

# ✅ DO：严格遵循力-色映射表，使用 create_force_arrow 工厂函数
from scripts.physics_graphics import create_force_arrow
# G=红(H0°) / f=黄(H51°) / F=绿(H142°) / N=蓝(H217°) / T=紫(H272°)
# 力学域内最小 ΔH=51°，工厂函数内部自动映射正确颜色
gravity = create_force_arrow(origin, [0, -1, 0], magnitude=9.8, label=r"\vec{G}")
friction = create_force_arrow(origin, [0.87, -0.5, 0], magnitude=4, label=r"\vec{f}")
normal = create_force_arrow(origin, [0, 1, 0], magnitude=9.8, label=r"\vec{N}")
```

**画面炸成**：颜色相近无法区分受力物体，物理关系混乱
