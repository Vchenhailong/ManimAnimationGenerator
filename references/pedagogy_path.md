# 教学路径设计规范（解决没路径的困境）

## 核心理念

每个动画视频必须为学习者提供一条清晰的认知路径，从已知到未知、从具体到抽象、从直观到形式化，最终让学习者自行构建理解。

## 1. 认知负荷控制

- 单视频时长 <= 8 分钟（约 80 个教学原子），超过则拆分为系列。
- 每个教学原子仅引入一个新概念或一个新操作。
- 语音文本避免嵌套从句，保持主谓宾清晰。

## 2. 必经的七个阶段（强制顺序）

对于任何概念，动画必须按以下顺序设计（可微调顺序，但不可跳跃缺失）：

| 阶段           | 目的             | 教学策略       | 示例（矩阵乘法）               |
| -------------- | ---------------- | -------------- | ------------------------------ |
| ① 激活已有知识 | 唤醒前置经验     | 提问或简短回顾 | 还记得两个向量的点积吗？       |
| ② 直观体验     | 建立感性认识     | 动画/生活类比  | 展示矩阵作用在猫图片上产生变形 |
| ③ 形式化定义   | 给出严谨表述     | 公式 + 关键词  | (AB)_{ij} = Σ A_{ik}B\_{kj}    |
| ④ 运算示范     | 展示如何应用规则 | 分步动画       | 逐行乘列计算具体数值           |
| ⑤ 反直觉点澄清 | 破解常见误解     | 先错后对       | 不是按位乘——先展示错误         |
| ⑥ 应用与迁移   | 用于解决问题     | 案例、跨学科   | 图像旋转、推荐系统             |
| ⑦ 总结与回顾   | 梳理核心要点     | 思维导图式     | 展示包含所有要点的摘要图       |

## 3. 直观优先原则

- 在任何形式化定义之前，必须有直观体验阶段（持续 >= 6 秒）。
- 直观体验使用动画、几何变换、颜色高亮、对比，禁止直接抛公式。

## 4. 反直觉点强制处理

- 每个常见反直觉点必须有专门步骤（阶段⑤）。
- 错误猜测用灰色文字/虚线框展示，再用正确动画覆盖。

## 5. 案例多样性要求

- 至少一个日常生活案例（如旋转照片）。
- 至少一个工程/科学案例（如计算机图形学）。
- 案例应参数化，并直观展示输入到输出的变换。

## 6. 学习者自检

- 视频结尾增加一个思考题或小测验语音提示（<= 10 秒）。

## 7. 路径可视化

- 视频开头展示一个路径图（时间线或节点图），让学习者知道将要经历哪些阶段。
- 每个阶段开始时，高亮路径图中的对应节点。
- 路径图最后淡出。

## 8. 因果链强制展示

- 每个定义或运算规则后，必须附上一个为什么这样的直观解释（时长遵循统一规范）。
- 反直觉点的证伪步骤必须包含原因分析（例如因为乘法不满足交换律，所以）。

## 9. 知识网络图

- 视频结尾展示一张图，包含当前概念、其前置知识、后续可能相关的概念，用箭头标注关系。
- 该图可使用 Manim 的 Graph 类绘制，并附带简短语音总结。

## 10. 讲解深度规范（解决不知其所以然的困境）

为确保每个知识点被讲清楚而非抛公式，必须遵循以下深度标准。

### 10.1 什么算讲清楚？

一个知识点被认为讲清楚的条件（全部满足）：

- 定义明确：给出严格的数学/科学定义，并解释每个符号的含义。
- 直观解释：用图形、动画或生活类比说明定义的含义。
- 至少一个例子：展示具体数值或简单场景下的应用。
- 与前置知识连接：明确说明这个新概念是基于什么旧知识。
- 反直觉澄清（如适用）：指出容易误解的地方并解释为什么。

### 10.2 什么情况下必须展开讲解？

以下情况不能只给公式或一句话带过，必须逐层展开：

- 首次引入的新概念、新定义。
- 容易与日常经验冲突的知识（如负负得正、概率不等于确定性）。
- 推导过程超过 2 步的公式。
- 学生常见错误高发点（如对数运算、矩阵乘法不交换）。

### 10.3 展开到何种程度？

使用三句法作为每个知识原子的最低讲解标准：

| 层次       | 内容           | 示例（矩阵加法）                                                                   |
| ---------- | -------------- | ---------------------------------------------------------------------------------- |
| ① 是什么   | 定义/陈述      | 矩阵加法是指两个形状相同的矩阵，对应位置上的元素分别相加。                         |
| ② 为什么   | 原因/推导/直觉 | 为什么可以这样做？因为矩阵加法的本质是多个线性方程组的并行计算，每个方程独立相加。 |
| ③ 举个例子 | 具体数值或场景 | 比如 A 的第一行第一列是 1，B 的第一行第一列是 5，相加就是 6。                      |

反例：

- 不展开：矩阵加法就是对应元素相加。（只有①是什么）

正例：

- 展开：①是什么 + ②为什么 + ③举个例子

### 10.4 与七阶段的关系

讲解深度规范是对七阶段中 ③ 形式化定义 和 ④ 运算示范 的细化要求：

- 阶段③ 形式化定义：必须包含定义明确 + 直观解释 + 与前置知识连接
- 阶段④ 运算示范：必须包含至少一个例子 + 分步展示

### 10.5 验收关联

在 verification_checklist.md 中已增加对应检查项：

- 讲解深度每个新概念满足：定义明确 + 直观解释 + 至少一个例子 + 连接前置知识 + 反直觉澄清（如适用）

## 11. 动画节奏与打字机效果规范

### 11.1 文字出现方式

- 所有文字内容必须使用打字机效果逐字/逐字母显示，增强学习者的逐字阅读感和节奏感
- 实现方式：AddTextLetterByLetter（纯文本）或 Write（公式）
- 禁止使用 FadeIn 整体淡入文字内容（标题、路径图等过渡性内容除外）

### 11.2 速度参数

- 打字机速度：0.05 秒/字符（中文）
- 英文字母：0.03 秒/字符
- 公式（MathTex）：按元素个数计算，每个元素 0.15 秒

### 11.3 元素间延迟

- 同一教学原子内的多个元素（如：矩阵 + a\_{ij} + 是元素）之间
- 延迟 0.2 秒，让学习者有时间视线移动

### 11.4 语音与打字机的同步

- 打字机动画应在语音开始后立即播放
- 动画时长应略短于语音时长（预留 0.3~0.5 秒缓冲）
- 禁止动画结束后长时间静默等待

### 11.5 示例代码

# 打字机效果

```python
self.play(AddTextLetterByLetter(text), run_time=0.05 * len(text))
self.wait(0.2)
```

# 公式逐步绘制

```python
self.play(Write(formula), run_time=0.15 * len(formula.submobjects))
self.wait(0.2)
```

## 12. 图形绘制与几何示意要求（新增）

### 12.1 图形类型与教学阶段的对应关系

| 教学阶段       | 推荐图形类型         | graphics.type            | 说明                   |
| -------------- | -------------------- | ------------------------ | ---------------------- |
| ② 直观体验     | 生活类比图、几何变换 | linear_algebra / polygon | 展示矩阵变换的几何意义 |
| ③ 形式化定义   | 坐标系、矩阵图形     | axes / polygon           | 展示矩阵的结构         |
| ④ 运算示范     | 分步动画、高亮图形   | linear_algebra           | 展示向量变换过程       |
| ⑤ 反直觉点澄清 | 对比图形、错误示例   | polygon / axes           | 展示不满足交换律的例子 |
| ⑥ 应用与迁移   | 实际场景图、物理示意 | physics / polygon        | 图像处理、机器人运动   |
| ⑦ 总结与回顾   | 知识网络图、思维导图 | graph / polygon          | 展示概念关系           |

### 12.2 图形绘制基本原则

1. 几何图形顶点必须由数学公式推导生成，禁止硬编码坐标
2. 图形区默认位于右侧（两栏/三栏模式），与主内容区顶部对齐
3. 图形应与当前教学阶段匹配，避免无关图形干扰
4. 图形应配合 visual_action 实现动画效果（高亮、变换、移动）

### 12.3 graphics 字段使用示例

矩阵乘法的直观体验阶段（阶段②）：
{
"type": "linear_algebra",
"params": {
"vectors": [
{"name": "i", "coords": [1, 0], "color": "RED"},
{"name": "j", "coords": [0, 1], "color": "GREEN"}
],
"transform": {
"matrix": [[2, 0], [0, 1]],
"show_animation": true
}
}
}

矩阵定义阶段（阶段③）：
{
"type": "polygon",
"params": {
"shape": "rectangle",
"width": 4,
"height": 3,
"center": [0, 0],
"color": "#66DDFF",
"fill_opacity": 0.2
}
}

应用阶段（阶段⑥）：
{
"type": "physics",
"params": {
"object": "robot_arm",
"position": [0, 0],
"angles": [0, 45, 90],
"show_animation": true
}
}
