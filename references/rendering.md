# 渲染规范

## 1. 相机配置

### 1.1 2D 场景

- 相机类型：正交相机（Orthographic）
- 位置：居中，正视画布
- 范围：X ∈ [-7.5, 7.5]，Y ∈ [-4.5, 4.5]
- 无透视变形，适合公式、图表、几何图形

### 1.2 3D 场景

- 相机类型：透视相机（Perspective）
- 距离：10 单位（相机位于 z = 10）
- 视角：45 度
- 初始位置：(3, 3, 5)
- 注视原点：(0, 0, 0)
- 支持 OrbitControls 交互（旋转/缩放）

### 1.3 相机切换规则

- 默认使用 2D 正交相机
- 当 graphics.type 为 three_d 时，自动切换为 3D 透视相机
- 不允许在同一场景中混合使用两种相机类型

## 2. 坐标系

### 2.1 原点位置

- 屏幕中心为原点 (0, 0, 0)
- 与 Manim 默认坐标系一致

### 2.2 坐标范围

- X 轴：-7.5 到 +7.5（总宽 15 单位）
- Y 轴：-4.5 到 +4.5（总高 9 单位）
- Z 轴：-5 到 +5（用于 3D 场景）

### 2.3 坐标方向

- X 轴：向右为正
- Y 轴：向上为正
- Z 轴：向屏幕外为正

### 2.4 多栏布局坐标系映射

| 布局模式 | 左栏 X 范围    | 中栏 X 范围  | 右栏 X 范围  | 备注                      |
| -------- | -------------- | ------------ | ------------ | ------------------------- |
| 单栏     | [-6.75, 6.75]  | —            | —            | 全安全区                  |
| 两栏     | [-6.75, 1.35]  | —            | [1.85, 6.75] | 有图形时左60%/右40%       |
| 三栏     | [-6.75, -2.2]  | [-1.7, 2.35] | [2.85, 6.75] | 有图形时左30%/中30%/右40% |
| 两栏对比 | [-6.75, -0.05] | —            | [0.55, 6.75] | 均分 50%/50%              |

## 3. 安全边距

### 3.1 固定边距

- 上下左右各保留 10% 安全边距
- 左边界：-6.75
- 右边界：+6.75
- 上边界：+4.0
- 下边界：-2.8（字幕区上边界）

### 3.2 字幕区

- 专用区域：Y ∈ [-3.85, -2.8]
- 高度：1.05 单位
- 最大行数：3 行
- 超出时垂直滚动，不显示滚动条

### 3.3 安全边距计算公式

- left = -7.5 + (15 × 0.1) = -6.75
- right = 7.5 - (15 × 0.1) = 6.75
- top = 4.5 - (9 × 0.1) = 4.05 → 4.0
- bottom = -4.5 + (9 × 0.1) = -3.6（但需为字幕留空间，实际取 -2.8）

## 4. 渲染分辨率

### 4.1 默认分辨率（推荐）

- **宽度**: 1280 像素
- **高度**: 720 像素
- **帧率**: 30 fps
- **用途**: 常规预览、教学视频发布、网络传输

> **说明**: 默认使用 720p 分辨率，兼顾画质清晰度与文件大小，适合大多数教学场景。

### 4.2 质量预设

| 预设          | 分辨率    | 帧率   | 用途                 |
| ------------- | --------- | ------ | -------------------- |
| 低质量（-ql） | 854×480   | 15 fps | 快速调试预览         |
| 中质量（-qm） | 1280×720  | 30 fps | 常规预览             |
| 高质量（-qh） | 1920×1080 | 30 fps | 高清展示（默认质量） |
| 4K（-qk）     | 3840×2160 | 60 fps | 超高清展示           |

配置定义见 `templates/manim.cfg`（`quality = medium_quality`）。

**为什么默认 720p？**

- 教学视频对分辨率要求适中，720p 已足够清晰
- 文件大小比 1080p 小约 50%，便于传输和存储
- 渲染速度更快，迭代效率更高
- 兼容大多数设备和播放平台

### 4.3 像素格式

- yuv420p（兼容所有播放器）

## 5. 输出格式

### 5.1 默认输出

- 格式：MP4
- 视频编码：libx264（H.264）
- 音频编码：aac
- 文件扩展名：.mp4

### 5.2 渲染命令

渲染命令格式：`manim -qh --disable_caching scene.py ClassName`

参数说明：

- `-qh`：高质量（1920×1080）
- `--disable_caching`：禁用缓存（语音同步必须）

渲染配置详见 `templates/manim.cfg`。

## 6. 帧率

### 6.1 默认帧率

- 30 fps（每秒 30 帧）
- 兼顾流畅度和文件大小

### 6.2 帧率适用场景

| 场景         | 推荐帧率 | 说明     |
| ------------ | -------- | -------- |
| 普通教学动画 | 30 fps   | 默认     |
| 高速运动演示 | 60 fps   | 物理模拟 |
| 纯文字/公式  | 24 fps   | 节省资源 |

## 7. 溢出处理策略

### 7.1 检测时机

- 在 construct 开始时虚拟测量所有元素的包围盒
- 在元素定位完成后进行安全边界检查

### 7.2 处理优先级

按以下顺序尝试，直到成功：

1. 整体缩放（字号不低于安全下限，参见 layout.md 优化策略）
2. 触发分栏升级（单栏 → 两栏 → 三栏）
3. 分组刷新（FadeOut 老旧内容组）
4. 拆分原子（将当前原子拆为多个）
5. 自动分场（拆分为多个场景）

### 7.3 兜底策略

当以上所有策略都失败时：

- 报错并终止渲染
- 输出详细错误信息：
  - 哪个原子（id）导致失败
  - 估算的内容宽度/高度
  - 建议的手动调整方案

错误信息示例：

```
[Render Error] 原子 'definition_formal' 无法安全渲染
预估高度: 6.8 单位 > 5.5 单位
预估宽度: 14.2 单位 > 13.5 单位
元素数量: 9 个 > 6 个
建议: 将该原子拆分为 2-3 个独立原子
```

## 8. 性能优化

### 8.1 渲染优化

- 使用 --disable_caching 避免缓存导致的语音不同步
- 复杂场景使用 -ql 快速预览，最终输出使用 -qh
- 避免创建超过 500 个 VMobject

### 8.2 内存管理

- 及时清理临时文件（media/ 目录）
- 使用 VGroup 合并多个元素，减少渲染开销

## 9. 分场与合并

### 9.1 分场触发条件

- 总原子数 > 30
- 单原子无法容纳（高度/宽度超出阈值）

### 9.2 分场命名规则

- 场景文件：主题*序号*子标题.py（如 matrix_01_definition.py）
- 内容文件：主题*序号*\_content.json（如 matrix_01_content.json）
- 类名：PascalCase，与子标题对应（如 MatrixDefinition）

### 9.3 合并命令

ffmpeg -f concat -safe 0 -i concat_list.txt -c copy full_video.mp4

concat_list.txt 格式：
file 'scene1.mp4'
file 'scene2.mp4'
file 'scene3.mp4'

### 9.4 目录结构示例

```
outputs/
├── matrix_01_definition.py
├── matrix_01_content.json
├── matrix_02_multiplication.py
├── matrix_02_content.json
├── matrix_03_inverse.py
├── matrix_03_content.json
└── concat_videos.sh
```

### 9.5 验证项

- 分场场景数量与原子数匹配（每场景 15-30 原子）
- 每个场景文件独立可渲染
- 合并脚本存在且命令正确
