# 字幕与字幕滚动机制

1. 字幕区的可见字幕，每次最多展示2行字幕，每行不超过20个字词（即以每行20字词进行截取）。
2. 字幕行展示的时间为4字词每秒的速度进行计算。每行字幕只会被计算一次，以避免重复计算而导致时间累加、字幕滚动错误。
3. 字幕按行联动滚动，要求字幕行滚动的时间一致、行间距一致。比如原子场景的字幕一共有85个字词，那么一共就是5行字幕；首次展示的是第1、2行字幕（其他行字幕隐藏）；滚动行时则第1行移出且隐藏（消失），第2行出现在原第1行，新的字幕行（原隐藏的第3行）出现在原第2行...以此类推，直至该场景最后一行展示在了字幕区。

# 主内容与字幕时长不一的情况下，场景时长与切换的原则

## 必须的原则

1. 哪个内容区（主内容区和字幕区）的内容所需总时间更长，则用哪个时间。对于存在对应的课程知识内容JSON，读取 `duration` 的值作为原子场景播放的时长后，遵循计算比较的原则。

场景示例说明：
例子1. 主内容区的渲染完成了，而此时字幕展示所需的时间（这是动态计算所得）相比主内容区渲染的耗时长，那么实际的场景时长就取字幕所需的时间。
例子2. 主内容区的渲染完成的时间假设是 N，对应的课程知识内容JSON原子场景的 `duration` 值为 A，而字幕展示所需的时间（这是动态计算所得）B 相比 A 更大（已知 N 比 A、B都小），那么实际的原子场景时长就取字幕所需的时间 B。
即 **总之取时间总耗时最长的为原子场景的时间**。

2. 字幕和主内容同时开始出现，但各自的速度由其内部控制

- 同时开始。主内容和字幕都在 t=0 时刻开始播放
- 各自速度 - 主内容自身设定与控制的时间内完成，字幕按 4.0 字/秒的原本速度

即 “一起跑，各自时控”
